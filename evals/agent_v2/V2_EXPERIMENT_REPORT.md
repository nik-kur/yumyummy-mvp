# Agent v2 Experiment — итоговый отчёт

**Ветка:** `exp/agent-v2` (изолирована; прод на Render деплоится только из `main`)
**Дата:** 2 июля 2026 · Все прогоны — локально, in-process (`evals/agent_v2/run_v2_eval.py`)

## Что построено

`app/agent_v2/` — экспериментальный движок «один интент → один вызов», без
router-ноды и без final-agent-ноды:

| Интент | Пайплайн | LLM-вызовов |
|---|---|---|
| `log_meal` (generic-еда) | parse (LLM) → USDA FDC lookup (параллельно) → детерминированная математика | 1 |
| `eatout` / `product` | один grounded-вызов (поиск внутри провайдера), max 3 запроса, JSON-схема | 1 |
| `photo_meal` | vision-вызов → (mode b) FDC-верификация generic-позиций | 1 |
| `barcode` | OpenFoodFacts напрямую, LLM-фолбэк только при промахе | 0–1 |
| `food_advice` | один grounded-вызов, 3 ранжированные опции | 1 |

Варианты (провайдер брендового/поискового вызова):
- **V2-G** — Gemini 3 Flash + Google Search grounding
- **V2-G35** — Gemini 3.5 Flash + grounding
- **V2-S** — Perplexity Sonar (поиск встроен)
- **V2-O** — OpenAI gpt-5-mini + web_search, reasoning=low (контроль «та же топология, тот же вендор»)
- **V1** — текущий прод (`app/agent_workflow`, gpt-5-mini medium, 5–10 поисков, 3 ноды) — запускался in-process без изменений

## Датасеты

- `brands_goldens_30_ref.csv` — 30 кейсов из мартовского эвала + замороженные reference-ккал (официальные значения) с допусками
- `brands_regions_v1.csv` — 14 новых кейсов: СНГ (Вкусно и точка, Додо, Теремок, ВкусВилл, Простоквашино, Агуша, Боржоми), Ближний Восток (Al Baik, Herfy, ливанское кафе), Баку (шах-плов), EU (Pret, PAUL), US (Sweetgreen)
- `generic_15.csv` — 15 generic-кейсов с reference из USDA (яйца, гречка, овсянка, борщ, сырники...)
- `barcodes_5.csv` — 4 реальных EAN + 1 несуществующий (тест фолбэка)

Метрики: латентность (median/p90), $/запрос (по фактическим токенам), domain-hit
(источник = ожидаемый бренд-домен, по citations провайдера), kcal-ok (в допуске
от reference), доля HIGH, живость source_url (реальный HTTP-чек — новая метрика,
которой в v1-эвалах не было).

Ключевое: V2-G **читает печатные цифры с этикеток/скриншотов** (промпт-правило
«printed numbers first») — сырники 272.7, окрошка 192.0, салат 202.8, скриншот
Яндекс Еды 440.8 и даже сумма двух продуктов на одном фото (371.7) совпали с
этикетками с точностью до десятых. Прод на тех же кейсах ходил в веб-поиск по
30–120 секунд и промахнулся сильнее (например, потерял курицу в фото с двумя
продуктами: 107 kcal вместо ~390).

Единственный промах V2-G — переоценка порции рассыпчатого творога на глаз
(502 vs ~280 kcal), классическая ошибка визуальной оценки веса, есть у всех
вариантов. Фото остаются `confidence=ESTIMATE` by design.

\* v1 не репортит usage; оценка из тарифов (5–10 поисков × $10/1k + mini-токены).
\*\* На бесплатной квоте grounding (5 000 промптов/мес на Gemini 3) фактически ~$0.004; $0.039 — после квоты ($14/1k поисковых запросов).
\*\*\* V2-G35: structured output отключает google_search — модель отвечает из памяти; наш guard честно даунгрейдит всё в ESTIMATE, но URL из памяти часто мёртвые. Не годится для брендов в текущем виде.

## Ключевые находки

1. **Топология решает.** Убрали router и final-agent, поиск ограничили 3
   запросами — латентность падает с 37–46s до 4–7s (в 6–9 раз) без потери
   качества. Даже на том же OpenAI (V2-O) выходит 12–15s — быстрее прода в 3
   раза, но медленнее Gemini/Sonar в 2–3 раза.
2. **Grounded-поиск провайдера ≫ агентский цикл WebSearchTool.** Один вызов с
   встроенным поиском находит официальные домены не хуже (73–75% vs 59–67%), при
   этом kcal-точность 96–100% против «не мерялось» в старом эвале.
3. **USDA FDC — бесплатный буст качества generic-еды.** 15/15 kcal в допуске
   (у прода 13/15), у каждой позиции ссылка на официальную базу, латентность
   4s. Композитные блюда (борщ, сырники) корректно уходят в LLM-оценку.
4. **Галлюцинированные ссылки — реальная проблема, и она решается.** Модели
   уверенно печатают битые deep-URL (у V2-G изначально 13/30 мёртвых). Резолвинг
   grounding-редиректов + подстановка URL из подтверждённых citations снизили до
   6/30 у V2-G и 3/30 у V2-S. У прода эта метрика вообще не мерялась.
5. **Gemini 3.5 Flash: grounding несовместим со structured output** (0 поисков
   на 30/30 кейсах; без схемы ищет нормально). Известное ограничение препро-
   модели — исключает V2-G35 из кандидатов для branded до фикса на стороне Google.
6. **Barcode без LLM — мгновенный.** 0.7s против 10.6s: gpt-4.1-нода в проде
   просто оборачивала тот же OFF-лукап.

## Рекомендация

**Основа: V2-G (Gemini 3 Flash + grounding) для branded/photo/advisor, parse→USDA для generic, OFF-директ для barcode.**
- Лучший domain-hit (73–75%), 100% kcal-ok, 90% HIGH, 6–7s median.
- **V2-S (Sonar) — второй провайдер**: самый быстрый и стабильный по латентности
  (4–5s, p90 5.4s), самые живые ссылки, $0.009/req фикс — идеален как фолбэк при
  429/деградации Gemini (интерфейс уже общий) или как primary, если приоритет —
  скорость и предсказуемость цены. Минус: чаще цитирует агрегаторы вместо
  офсайта и консервативнее с HIGH (63%).
- V2-O — доказал, что дело в топологии, но по всем метрикам хуже: в прод не берём.

Экономика на 1000 активных юзеров (~15 запросов/день, микс 60% generic / 25%
photo / 10% branded / 5% прочее): v2 ≈ **$2–4/юзер/год** против ~$5–8 у прода
(и это до кэша повторных запросов — «вторая волна»).
